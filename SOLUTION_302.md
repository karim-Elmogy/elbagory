# حل مشكلة 302 Found عند الوصول إلى لوحة التحكم

## المشكلة:
عند محاولة الوصول إلى `http://elbagory.test/admin/dashboard` تحصل على:
- **Status Code:** `302 Found`
- **Location:** `http://elbagory.test/login`

## السبب:
المستخدم **غير مسجل دخول**. الـ Middleware `auth` يعيد التوجيه إلى صفحة تسجيل الدخول.

## الحل:

### الخطوة 1: سجل الدخول أولاً

1. اذهب إلى صفحة تسجيل الدخول:
   ```
   http://elbagory.test/login
   ```

2. أدخل بيانات الدخول:
   - **البريد الإلكتروني:** `admin@example.com`
   - **كلمة المرور:** `password`

3. اضغط على "تسجيل الدخول"

### الخطوة 2: بعد تسجيل الدخول

بعد تسجيل الدخول بنجاح، يمكنك:
- الوصول إلى لوحة التحكم من القائمة العلوية (رابط "لوحة التحكم")
- أو اذهب مباشرة إلى: `http://elbagory.test/admin/dashboard`

## التحقق من البيانات:

تم التحقق - المستخدم موجود في قاعدة البيانات:
- ✅ **الاسم:** مدير النظام
- ✅ **البريد:** admin@example.com
- ✅ **الدور:** admin (مدير النظام)

## ملاحظات:

- الـ `302 Found` هو سلوك طبيعي عندما تحاول الوصول إلى صفحة محمية بدون تسجيل دخول
- بعد تسجيل الدخول، سيتم حفظ Session وسيمكنك الوصول إلى لوحة التحكم
- إذا نسيت كلمة المرور، يمكنك إعادة إنشاء المستخدم:
  ```bash
  php artisan tinker
  ```
  ```php
  $user = App\Models\User::where('email', 'admin@example.com')->first();
  $user->password = bcrypt('password');
  $user->save();
  ```
